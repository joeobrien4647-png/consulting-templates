> **A responsibility assignment framework for clear accountability and decision-making**

## üìã How to Use This Template

The RACI Matrix eliminates confusion about who does what. Use it to:

- **Clarify roles** ‚Äî Everyone knows their responsibility
- **Prevent gaps** ‚Äî No task falls through the cracks
- **Avoid duplication** ‚Äî Reduce wasted effort
- **Speed decisions** ‚Äî Clear escalation paths
- **Onboard faster** ‚Äî New team members understand quickly

### Instructions

1. **List all activities** ‚Äî Be specific, not generic
2. **Identify all roles** ‚Äî Functions, not individuals (individuals change)
3. **Assign RACI codes** ‚Äî One R per activity; C and I can be multiple
4. **Validate with stakeholders** ‚Äî Get buy-in before publishing
5. **Review quarterly** ‚Äî Update as the organization evolves

## üéØ RACI Definitions

| Code | Role | Definition | Rules |
|------|------|------------|-------|
| **R** | Responsible | Does the work | Only ONE per activity |
| **A** | Accountable | Ultimately answerable; has veto | Only ONE per activity |
| **C** | Consulted | Provides input before decision | Two-way communication |
| **I** | Informed | Notified after decision | One-way communication |

### Extended RACI (RASCI/RACI-VS)

| Code | Role | When to Use |
|------|------|-------------|
| **S** | Supportive | Provides resources or support to R |
| **V** | Verify | Checks work meets quality standards |
| **O** | Omitted | Intentionally not involved |

## ‚ö†Ô∏è Common RACI Problems & Solutions

### Red Flags to Watch For

| Problem | Symptom | Solution |
|---------|---------|----------|
| **No R** | Task has no one doing the work | Assign someone; if no one can, task may not be needed |
| **Multiple R's** | Confusion about who does what | Split activity into subtasks OR pick one R + one S |
| **No A** | No accountability; issues escalate poorly | Every activity needs ONE decision-maker |
| **Multiple A's** | Decisions stall; everyone waits | Only one A; others become C |
| **Too many C's** | Decisions take forever | Limit C to those with must-have input; move others to I |
| **R without authority** | Person blocked by lack of access/power | Ensure R has resources or add S roles |
| **A too far from work** | Accountability without understanding | A should be close enough to make informed decisions |

### Validation Checklist

For each activity:
- [ ] Exactly ONE R (Responsible)
- [ ] Exactly ONE A (Accountable) ‚Äî can be same as R
- [ ] C's are truly needed (two-way communication required)
- [ ] I's are appropriately limited
- [ ] R has authority/resources to complete
- [ ] A has authority to make final decision

For each role:
- [ ] Not too many R's (capacity check)
- [ ] A's are appropriate seniority
- [ ] Balance across roles (no one overwhelmed)

## üîß RACI Workshop Facilitation Guide

### Workshop Agenda (2 hours)

| Time | Activity | Output |
|------|----------|--------|
| 0:00-0:15 | Introduction & RACI overview | Shared understanding |
| 0:15-0:45 | Brainstorm activities | Activity list |
| 0:45-1:00 | Define roles | Role list |
| 1:00-1:30 | Assign RACI (group exercise) | Draft matrix |
| 1:30-1:45 | Validate & discuss conflicts | Agreed matrix |
| 1:45-2:00 | Define governance & review cycle | Operating rhythm |

### Facilitation Tips

1. **Start with activities, not roles** ‚Äî Avoids territorial behavior
2. **Use sticky notes** ‚Äî Easy to move and adjust
3. **Challenge every C** ‚Äî "Do they really need to be consulted?"
4. **Test with scenarios** ‚Äî "When X happens, who does what?"
5. **Document disagreements** ‚Äî Revisit after the meeting
